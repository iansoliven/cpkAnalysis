Command-Line Reference

The cpkanalysis.cli module provides subcommands for scanning STDF directories, executing the analysis pipeline, moving template data, and opening the post-processing menu.

Commands

  • scan — Recursively discover STDF files and write a metadata JSON manifest.
  • run — Execute the full pipeline (ingestion through metadata) with optional post-processing.
  • move-template — Copy the CPK Report sheet into another template sheet.
  • post-process — Launch the interactive post-processing menu for an existing workbook.

Run Command Options

Option Description
inputs One or more STDF file paths. If omitted, the tool scans the current directory.
--metadata path.json Load STDF file list from a JSON manifest created by scan.
--output path.xlsx Destination workbook (default CPK_Workbook.xlsx).
--template path Excel template file or directory containing a single template workbook.
--template-sheet name Target sheet to receive CPK data when copying into templates.
--outlier-method {none,iqr,stdev} Outlier filtering strategy (default none).
--outlier-k value Multiplier for IQR or standard deviation filters (default 1.5).
--no-histogram, --no-cdf, --no-time-series Skip generating the corresponding chart families to reduce runtime.
--plugin <directive> Enable, disable, reprioritise, or amend plugin parameters (e.g. enable:builtin.summary_logger).
--plugin-profile path.toml Override the default plugin profile (defaults to post_processing_profile.toml in the workspace).
--validate-plugins Run the analysis into a temporary workbook and discard outputs after plugin validation completes.
--postprocess Automatically launch the post-processing menu after the workbook is written.
--debug Print additional stack traces and environment details when diagnosing pipeline issues. (Primarily for development builds.)
--quiet Minimise console chatter by suppressing status messages (errors still print). Useful when wrapping the CLI in automation scripts.
--session-dir path Override the default temp/session_* directory for intermediate Parquet files. Handy when running multiple analyses simultaneously.
--skip-template Run ingestion, outlier filtering, statistics, and metadata generation without building the Excel template (produces a minimal workbook).

Examples

Basic Run

python -m cpkanalysis.cli run Sample/lot2.stdf Sample/lot3.stdf \
  --output temp/CPK_Workbook.xlsx

Using Metadata Manifest

python -m cpkanalysis.cli scan Sample --metadata temp/lot_manifest.json
python -m cpkanalysis.cli run --metadata temp/lot_manifest.json --postprocess

Post-Processing Only

python -m cpkanalysis.cli post-process --workbook temp/CPK_Workbook.xlsx

Validation Mode Tips

  • Temporary outputs are stored under temp/validation_* and deleted automatically.
  • Combine with plugins to spot-check post-processing behaviour without touching production workbooks.
  • The metadata JSON still records action logs even when validation mode is enabled.
